    • 1. Ik kan uitleggen wat NodeJS, Express en Liquid zijn, hoe die samenwerken, en hoe ik daarmee lokaal een server kan maken🤓
    • 2. Ik weet wat het doel van package.json is, heb met npm packages geïnstalleerd en weet hoe ik die gebruik in mijn server code🤓
    • 3. Ik kan met commando’s in de terminal een NodeJS project starten en stoppen, en kan uitleggen wanneer herstarten nodig is🤓
    • 4. Ik heb een strategie voor debuggen in NodeJS, en weet hoe ik console.log() in kan zetten en uit kan lezen op de server🫣

    • 7. Ik weet wat de request en response argumenten zijn binnen een route callback functie in Express, en welke ik waarvoor kan gebruiken😅
    • 8. Ik weet wat een query parameter is, hoe ik die kan gebruiken in server-side code, en hoe ik die via links en formulieren kan veranderen😅
    • 9. Ik snap wat HTTP is, kan het verschil tussen GET en POST methods uitleggen, en snap dat mijn NodeJS code zowel server als client is😅
    • 10. Ik weet wat een REST API is, wat valide JSON data is en kan JSON data fetchen van verschillende API endpoints😅
    • 11. Ik kan data uit een REST API filteren, sorteren of doorzoeken door query parameters aan te passen😅
    • 12. Ik kan uitleggen wat Liquid doet, wat een view is, wat server-side rendering inhoudt en hoe ik data aan een view meegeef om dynamische HTML te renderen😅
    • 13. Ik weet hoe ik in Liquid partials kan maken en gebruiken, en data mee kan geven aan partials😅
    • 14. Ik weet hoe ik verschillende Liquid filters toe kan passen en waar ik kan vinden hoe ze werken😅
    • 15. Ik heb een strategie voor het debuggen van dynamische data in Liquid views met het json filter🫣
    • 16. Ik weet wat control flow in Liquid is, en hoe ik if en for loops kan gebruiken in een Liquid view🫣
    • 17. Ik weet wat variabelen, objects, functions, keywords, arrays, strings, statements en expressions zijn in JavaScript🫣
    • 18. Ik snap wat de async en await keywords doen in JavaScript code, en wat die te maken hebben met promises🫣
    • 19. Ik begrijp het verschil tussen client-side JS en server-side JS en kan uitleggen waarom JS in browsers onbetrouwbaarder is dan NodeJS🫣
    • 20. Ik kan gebruikers via een formulier data met een POST naar mijn server laten sturen, dit bewaren in een variabele en gebruiken in views😅
    • 21. Ik kan naar de server gePOSTe formulier data bewaren in een REST API en weet die weer terug te vinden via een filter😅
    • 22. Ik weet hoe ik Content First en mock data in kan zetten om snel een HTML prototype te maken met server-side code🫣
    • 23. Ik weet hoe ik verschillende UI states uit de UI stack kan verwerken in mijn views, en hoe ik daarin feedforward en feedback mix🫣
    • 24. Ik weet dat ik niet weet hoeveel verschillende devices, operating systems, browsers, versies en varianten daarvan er in gebruik zijn en wat dat betekent voor testen🥞
    • 25. Ik weet wat Progressive Enhancement betekent, en ken minimaal vijf strategieën om dat toe te passen in mijn HTML, CSS en JS, waaronder feature detection🫣
